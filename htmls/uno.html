Pruebas de aceptación

Historia 1: Página de inicio de sesión

  • ●Verificar información de nuevo usuario. 

    • ○El sistema debe verificar que la información que ingresó el usuario sea correcta. 

  • ●Verificar que el usuario no tenga cuenta. 

    • ○El sistema deberá de consultar con la base de datos para verificar si ese usuario existe y evitar que genere otra cuenta. 

  • ●Tipos de inicio de sesión. 

    • ○El sistema debe de mostrar distintos tipos de inicio de sesión dependiendo al perfil del usuario. 

 

Historia 2: Página de administración de cuenta

  • ●Estado de cuenta. 

    • ○El usuario podrá ver su estado de cuenta como primera vista. 

  • ●Pedidos. 

    • ○El usuario podrá ver sus pedidos como segunda vista y poder cancelarlos ahí directamente. 

  • ●Ocultar información delicada. 

    • ○La información del usuario deberá estar oculta para evitar que personas lo vean mientras navega por la plataforma. 

  • ●Verificar correo. 

    • ○Mostrar un mensaje al usuario si este no ha verificado su correo. 

  • ●Cambio de correo y reenvío de verificación. 

    • ○El sistema deberá reenviar un correo de verificación si el usuario cambia su correo. 

 

Historia 3: Surtir pedido de compras al proveedor

  • ●Solo productos pagados. 

    • ○El sistema solamente mostrará los productos que el usuario ya haya pagado. 

 

Historia 4: Administración de stock de producto

  • ●Estado de los productos. 

    • ○El sistema deberá mostrar el estado actual de los productos: vendido, en stock, agotado, etc. 

  • ●Aviso de agotado. 

    • ○Debe de existir un apartado específicamente para los productos que ya se han agotado para su rápida exploración. 

  • ●Automatización de pedidos. 

    • ○El encargado del almacén podrá automatizar pedido de stock dependiendo a que si el producto ya se agotó o si hay una cantidad específica o rango designado por él. 

  • ●Información del producto. 

    • ○El sistema deberá obligar al encargado de ingresar una descripción, precio, etc. del producto. 

 

Historia 5: Generar inventario

  • ●Filtros. 

    • ○El encargado de almacén podrá elegir como filtrar el stock al momento de generar el inventario: más vendidos, menos vendidos, por mes, descendente fecha, ascendente fecha, alfabéticamente, etc. 

  • ●Diferentes tipos de archivo. 

    • ○El encargado de almacén podrá elegir distintos tipos de archivos que desea: xlsx, doc, pdf, csv. 

  • ●Generación automática de inventario. 

    • ○El sistema, cada mes, se encargará de generar el inventario y ponerlo a disposición de las personas encargadas. 

 

Historia 6: Generar notas de venta

  • ●Generación automática. 

    • ○El sistema creará las notas de venta automáticamente cada mes y ese será enviado a todos los trabajadores de la empresa. 

  • ●Gráficas. 

    • ○El sistema debe de inyectar gráficas que describen el número de ventas, pérdidas, etc. 

  • ●Descripción mensual automáticamente. 

    • ○El sistema deberá de generar texto automáticamente que describa el estado actual de la empresa en cuanto a ventas dependiendo al estado de la venta. 

  • ●Descripción de clientes. 

    • ○El sistema deberá mostrar información de los distintos perfiles de los clientes: nuevos, compradores, vendedores, socios, etc. 

  • ●Venta por cliente. 

    • ○El sistema deberá mostrar distintos perfiles de cliente dependiendo a su compras, actividad, lugar, etc. 

 

Historia 7: Generar facturas

  • ●Generación de notas solamente si se compró el producto 

    • ○El sistema generará las notas de ventas al usuario, solo si este ya ha realizado su pago correspondiente. 

  • ●Información de la nota de venta. 

    • ○La información que debe de tener la nota de venta, debe corresponder a la información que pide el SAT. 

  • ●Formato de la nota de venta. 

    • ○El usuario podrá elegir los siguientes formatos: png, jpg, pdf. 

  • ●Recepción de nota de venta. 

    • ○La nota de venta se enviará al correo que el usuario ingresó.
